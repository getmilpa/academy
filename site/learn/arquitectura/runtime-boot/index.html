1Entender

El boot real empieza con una compuerta: el kernel refleja los manifiestos y resuelve el grafo completo a través de milpa/resolver ANTES de que cualquier plugin arranque. Un grafo bloqueado lanza ArchitectureBlockedException con el ResolutionReport completo a bordo — cada error aprendible, cada conflicto, cada ausencia — no solo un mensaje de una línea.

Un grafo booteable arranca en el orden que el propio reporte trae: loadOrder[]. La misma resolución que validó el grafo también lo ordenó, así que el orden de boot no es una lista aparte que pueda divergir; los registries y las rutas se derivan después, de un conjunto que ya demostró cerrar.
