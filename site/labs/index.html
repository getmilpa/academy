    Las cuatro prácticas

    1. 01 Arranca una Milpa real

      Crear el skeleton, arrancar su kernel y reconocer en la salida las piezas que efectivamente quedaron conectadas.

    2. 02 Rompe y repara un contrato

      Observar que requires/provides falla antes del boot y comprobar que agregar un provider repara el grafo.

    3. 03 Genera e inspecciona una ruta

      Usar devtools para generar código y después comprobar la ruta desde el kernel arrancado, no desde el filesystem.

    4. 04 Activa la superficie agent-ready

      Habilitar MCP de forma explícita, generar una tool y comprobar que el runtime la registró con nombre y descripción.
